# Xray конфигурации

- **config-reality.json** — VLESS + Reality (подмена под реальный TLS). Замените:
  - `REPLACE_WITH_USER_UUID` — на сервере можно оставить один UUID для всех клиентов (тогда Backend выдаёт один и тот же UUID в конфигах для этого сервера) или использовать динамическое добавление клиентов через API Xray (сложнее). В типовой схеме Backend генерирует конфиг с UUID пользователя; на ноде принимаются все подключения по одному или нескольким UUID (см. документацию Xray).
  - `REPLACE_WITH_REALITY_PRIVATE_KEY` — приватный ключ Reality (x25519).
  - `REPLACE_WITH_SHORT_ID` — shortId (например, `openssl rand -hex 4`).

- **config-tls.json** — VLESS + TLS с вашим сертификатом. Положите fullchain.pem и privkey.pem в `/etc/xray/`.

На production нодах обычно используют **один общий UUID на ноду** или **динамическое добавление users** через внешний конфиг/API, а Backend при выдаче конфига подставляет адрес сервера и этот UUID. Либо каждый пользователь имеет свой UUID в БД — тогда на Xray нужно либо перезагружать конфиг при добавлении пользователя, либо использовать Xray API для добавления inbound users.

Упрощённая схема: одна нода — один UUID; в БД хранится mapping user_id → server_id; конфиг для пользователя = выбранный сервер + общий для этой ноды UUID (если политика «один UUID на ноду»). Альтернатива: каждый user имеет свой UUID в БД, на нодах при старте подгружается список UUID из Backend (скрипт генерирует config с массивом clients).
