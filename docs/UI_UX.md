# UI/UX — Мобильное приложение и веб

## Стиль

- **Минимализм**: мало элементов, акцент на одну главную кнопку (Connect/Disconnect).
- **Тёмная и светлая тема**: переключатель в настройках; системная тема по умолчанию.
- **Современный интерфейс**: скруглённые карточки, достаточные отступы, читаемая типографика.
- **Плавные анимации**: переход подключения (пульс, индикатор), смена экранов, появление списка серверов.

## Главный экран

- **По центру**: большая круглая или каплевидная кнопка **Connect** (в состоянии «подключено» — **Disconnect**, цвет меняется на акцентный/красный).
- **Над кнопкой**: текущий выбранный сервер (название + регион), например «Netherlands 1 • EU». Тап — переход к выбору сервера.
- **Под кнопкой**: статус (Disconnected / Connecting / Connected), опционально пинг в мс и IP (после подключения).
- **Сверху**: логотип, справа — иконка настроек (или профиль). В настройках: тема, учётная запись, подписка, выбор региона по умолчанию (Auto / конкретный).
- **Снизу или в выдвижной панели**: кнопка «Выбрать сервер», «Подписка», «Настройки».

## Экран выбора серверов

- Список регионов (EU, US, Asia, …) с группами серверов внутри.
- Для каждого сервера: название, флаг/иконка региона, пинг (обновляется при открытии экрана или по таймеру). Сортировка: по пингу или по региону.
- Опция «Автоматический выбор» (лучший по пингу) вверху списка.
- Плавная анимация появления списка и обновления пинга.

## Экран подписки

- Текущий статус: активна до DD.MM.YYYY или «Нет активной подписки».
- Планы: 1 месяц, 3 месяца, 12 месяцев — карточки с ценой и выгодой (например, «−40%» для годовой).
- Кнопки «Оформить» ведут на Apple IAP / Google Play Billing.
- После покупки — экран успеха и возврат на главный экран; подписка подтягивается с backend (GET /api/subscribe).

## Защита от утечки DNS

- В настройках опция «DNS через туннель» (включена по умолчанию): весь DNS-трафик идёт через VPN. В Xray это настраивается в конфиге (routing + outbound DNS или fakedns). В приложении — использовать конфиг, сгенерированный backend с учётом этого (при необходимости backend отдаёт параметры DNS в конфиге).

## Уникальность дизайна

- Избегать «типового» фиолетового градиента. Варианты: тёмно-синий + бирюзовый акцент; тёмно-серый + оранжевый/зелёный; монохром с одним ярким акцентом на кнопке Connect.
- Шрифт: один акцентный (заголовки) + один нейтральный (текст), например SF Pro / Roboto или что-то вроде Manrope / Outfit для современного вида.
- Иллюстрации: минимальные (один экран «пустое состояние» при отключённой подписке), без перегруза.
