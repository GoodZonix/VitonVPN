# Безопасность и анти-абьюз

## Защита API

- **JWT**: доступ к `/api/config`, `/api/subscribe` только с валидным токеном. Срок жизни токена — 7 дней.
- **Rate limiting**: Redis-based лимит запросов по IP (например, 100 req/min на публичные эндпоинты). Для логина/регистрации — отдельный лимит (например, 10/мин на IP).
- **CORS**: настроить разрешённые origin только на домены вашего фронта и мобильных приложений (при необходимости).
- **HTTPS**: в production обязателен TLS для API и админки.

## Защита конфигов

- Конфиг VLESS выдаётся только авторизованному пользователю с активной подпиской.
- Не светить конфиги в публичных URL (например, не делать ссылку вида `/config/xxx` без авторизации).
- Короткий TTL ссылки на конфиг не обязателен: конфиг привязан к UUID пользователя; при компрометации — смена UUID в БД и перевыпуск конфига.

## Анти-абьюз

- **Лимит устройств**: в таблице `devices` — не более N записей на user_id (например, 5). При регистрации нового устройства сверх лимита — либо отказ, либо замена самого старого (по last_seen).
- **Проверка подписки**: при каждом запросе конфига проверять `subscriptions.expires_at`. Если подписка истекла — возвращать пустой `vless_urls`, мобильное приложение отключать VPN.
- **Отключение VPN при истечении подписки**: клиент периодически вызывает GET /api/subscribe или GET /api/config; при `active: false` — разрывать туннель и показывать экран продления.

## Ограничение количества устройств

- При логине/регистрации передаётся `device_id`. Устройство добавляется в `devices` (upsert).
- Перед выдачей конфига проверять: количество устройств по user_id ≤ MAX_DEVICES. Если превышен — не выдавать конфиг или возвращать ошибку с сообщением «удалите устройство в настройках».
- В админке — возможность просмотра и отзыва устройств пользователя.

## Рекомендации

- Логировать подозрительную активность (много неудачных логинов, смена IP, массовая выдача конфигов).
- Хранить пароли только в виде bcrypt-хеша (реализовано в backend).
- Секреты (JWT_SECRET, ключи Reality, DB URL) — только в переменных окружения или секрет-менеджере, не в коде.
